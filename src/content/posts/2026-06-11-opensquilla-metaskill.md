# OpenSquilla MetaSkill：技能编排的最后一块拼图

> 原文作者：花叔 | 整理：陈夏亮 | 2026-06-11

---

## 背景：三块拼图

花叔这两个月一直在折腾 Agent Skill，做了两个系统：

| 系统 | 功能 | 类比 |
|------|------|------|
| **女娲** | 把人的思维方式蒸馏成 Skill | 技能「生出来」 |
| **达尔文** | 让 Skill 自动迭代优化 | 技能「变好」 |
| **？** | 把多个 Skill 组织成流水线 | 技能「串起来」 |

问题是：**技能造得越多越乱，Agent 不知道怎么把它们串起来用。**

配一个写作流程，得自己想清楚先搜资料→学风格→写初稿→审校，一步步写死。第三块拼图一直没拼上。

直到发现了 GitHub 上 2.4k+ 星的开源项目 **OpenSquilla 3.0** 里的 **MetaSkill**。

---

## OpenSquilla 是什么

> 和 OpenClaw、Hermes Agent 同赛道，但走了几步不同的路。

核心理念：**不卷模型，卷 Harness。**

- **模型** = 发动机
- **Harness** = 变速箱 + 油门 + 导航

别人比发动机马力，它优化外面那层壳：每轮该调哪个模型、用哪个技能、怎么不白烧 token。

---

## 两个关键发现

### 1. 自动认领已有技能

装好后打开技能列表，它认出了 **134 个技能**。

没手动导入——它自动扫描到了 Claude Code 里的个人技能：飞书、浏览器自动化、女娲、达尔文等，全部原样可用。

> 社区 Skill 爆发，但每个人真正用得顺手的是自己攒的那几个。不用从零重配，这一点很省心。

### 2. 一句话造出新技能（MetaSkill Creator）

测试需求：

> "先对中文初稿做事实核查，再改写成口语化、去 AI 腔的版本，最后输出修改清单。"

生成结果——一张完整的流程图：

```
fact_check（事实核查） → rewrite（去AI腔改写） → modlist（生成修改清单）
```

每一步用哪个技能、依赖前面哪一步、结果传给谁，全部自动填好。手写要半小时，现在几分钟搞定。

---

## MetaSkill 到底是什么

### 大白话解释

**本质**：一份特殊的 Markdown 文件，标记为 meta 技能，下面把步骤连成流程图。

**关键区别**：

| 传统编排 | MetaSkill |
|----------|-----------|
| 我提前写死流程，Agent 照着跑 | 我只说目标，Agent 自己现编流程 |
| 我画好路线图 | 我给地图和规则，Agent 自己找路 |

### 安全护栏

不是放任 Agent 乱来。流程图交给 runtime 后：

- ✅ 依赖顺序强制校验
- ✅ 工具白名单
- ✅ 风险等级
- ✅ 模板安全
- ✅ 每步能读文件/跑命令都得提前声明

> 既让它自己组织，又给它上了护栏。

### 一句话总结

> 以前是我把技能怎么配想清楚再喂给 Agent；现在是我把目标说清楚，Agent 自己把技能配出来。

## 更省钱的点：智能模型路由

OpenSquilla 1.0 的底座能力——**自动选模型**。

### 测试场景

| 任务 | 路由选择 | 说明 |
|------|----------|------|
| "用一句话解释 HTTP" | 最便宜的 mini 模型 | 不开思考，成本最低 |
| 狼羊白菜过河 + 图着色分析 | 最强 code 模型 + 深度思考 | 自动升级 |

> 两档输入价格差约 **16 倍**。

### 跨厂商路由

用 OpenRouter 测试：
- 简单题 → DeepSeek flash（便宜）
- 难题 → Claude Opus + 深度思考（最强）

**简单活给国产便宜模型，难活才上旗舰，选择跨厂商完成。**

### 25 个任务的总账

和单用 Opus 方案比：**分数基本无差异，花费差出一个数量级。**

### 核心观点

> 几乎所有 Agent 产品都把选模型甩给用户：下拉菜单一排模型自己挑。普通用户凭什么要懂这些？让人为每次对话操心该选哪个模型，本身就是产品没做好。自动路由把这份负担免掉，你只管说要干什么。

---

## 总结：三块拼图凑齐

```
造（女娲） → 优化（达尔文） → 组织（MetaSkill）
```

| 阶段 | 工具 | 解决的问题 |
|------|------|-----------|
| 造技能 | 女娲 | 把思维方式蒸馏成 Skill |
| 优化技能 | 达尔文 | 让 Skill 自动迭代变好 |
| 组织技能 | MetaSkill | 把多个 Skill 串成流水线 |

**当技能多到一定程度，瓶颈不是模型聪不聪明，而是技能能不能被组织起来。**

OpenSquilla 的答案：把「组织」做进 Agent 底座，让 Agent 自己来，而不是等人一条条配。

> 行业注意力都在「模型又涨了几分」上。但工作流编排，可能正在变得比参数量更重要。
